# Local Copilot Chat

GitHub Copilot 계정으로 로그인해 로컬 웹 화면에서 채팅을 사용할 수 있는 앱입니다.

이 앱은 브라우저에서 GitHub 로그인만 완료하면 바로 사용할 수 있고, 대화는 서버의 로컬 DB에 저장되어 같은 계정으로 다시 로그인했을 때 이어서 볼 수 있습니다.

## 무엇을 할 수 있나요?

- GitHub Copilot 계정으로 로그인
- 여러 개의 대화창 생성 및 전환
- 스트리밍 방식으로 답변 받기
- 모델 선택 후 대화하기
- 같은 계정으로 다시 로그인하면 이전 대화 복원
- 같은 서버를 보는 다른 기기에서도 같은 계정 대화 복원
- 최근 1주일 이내 대화만 서버에 보관

## 시작하기

필요한 것:

- Python 3.10 이상
- GitHub Copilot을 사용할 수 있는 GitHub 계정

의존성 설치:

```bash
pip install -r requirements.txt
```

`.env` 파일은 없어도 실행할 수 있지만, 대부분의 경우 아래 값은 넣는 것을 권장합니다.

```dotenv
COPILOT_ENVELOPE_SECRET=replace-with-long-random-secret
```

이 값이 없으면 서버를 재시작할 때 다시 로그인해야 할 수 있습니다.

앱 실행:

```bash
uvicorn main:app --reload
```

브라우저에서 아래 주소를 엽니다.

```text
http://127.0.0.1:8000
```

## 사용 방법

1. 페이지를 열면 로그인 상태를 확인합니다.
2. Copilot 버튼을 눌러 GitHub 로그인을 시작합니다.
3. 열리는 GitHub 화면에서 코드를 입력해 인증합니다.
4. 로그인 완료 후 원하는 모델을 선택합니다.
5. 메시지를 보내면 답변이 실시간으로 표시됩니다.

## 대화 보관 방식

- 대화는 서버의 로컬 SQLite DB에 저장됩니다.
- 로그인된 대화는 GitHub 계정 기준으로 보관됩니다.
- 로그아웃해도 서버에 있는 대화는 지워지지 않습니다.
- 같은 계정으로 다시 로그인하면 이전 대화를 다시 볼 수 있습니다.
- 같은 서버와 같은 DB를 사용하는 다른 기기에서도 같은 계정 대화를 볼 수 있습니다.
- 1주일이 지난 대화는 서버에서 자동으로 정리됩니다.

## 로그인과 보안

- 브라우저에는 암호화된 로그인 정보만 저장됩니다.
- GitHub access token이나 Copilot token 평문은 브라우저에 직접 저장되지 않습니다.
- 현재 브라우저 세션과 맞지 않는 자격정보는 사용할 수 없도록 보호됩니다.
- 로그아웃하면 현재 브라우저는 다시 인증이 필요합니다.

## 사용량 표시

사용량 정보는 GitHub가 내려주는 내부 응답을 서버가 가능한 범위에서 읽어 화면에 보여줍니다.

- 로그인은 성공했더라도 사용량 표시는 일부만 보일 수 있습니다.
- Free 계정은 Chat 사용량만 보이고 Premium 영역은 숨겨질 수 있습니다.
- Pro 계정은 Premium requests와 무제한 Chat 상태가 함께 보일 수 있습니다.
- 사용량 표시는 참고용이며, 표시가 일부만 되더라도 채팅 자체는 계속 사용할 수 있습니다.

## 모델 선택

- 화면에는 사용하기 쉬운 이름의 모델 목록이 표시됩니다.
- 서버는 실제 Copilot API 호출에 필요한 모델 ID를 따로 사용합니다.
- 허용된 모델만 선택할 수 있습니다.

## 검색 기능

- 일반 질문은 그대로 Copilot에 보냅니다.
- 사용자가 명시적으로 검색을 요청한 경우에만 웹 검색을 보조로 사용합니다.

## 알아둘 점

- 이 앱은 개인 또는 소규모 내부 도구 용도에 맞춘 로컬 실행 앱입니다.
- 사용량 정보는 GitHub 내부 응답 형식에 의존하므로, GitHub 변경 시 일부 표시가 달라질 수 있습니다.
- 대화 복원은 같은 서버와 같은 로컬 DB를 사용하는 환경을 기준으로 동작합니다.
